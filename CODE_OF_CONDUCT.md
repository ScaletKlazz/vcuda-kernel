# Code of Conduct

## Our Pledge

We are committed to making participation in this project respectful,
professional, and focused on technical progress. Contributors should be able to
discuss kernel, GPU, Kubernetes, and virtualization work without harassment or
personal attacks.

## Expected Behavior

- Be respectful and direct.
- Keep technical criticism focused on code, design, evidence, and risk.
- Assume good intent when reviewing patches or bug reports.
- Provide reproducible details when reporting failures.
- Accept maintainer decisions after discussion has reached a clear conclusion.

## Unacceptable Behavior

- Harassment, insults, threats, or discriminatory language.
- Personal attacks in code review or issue discussion.
- Publishing private information without explicit permission.
- Repeated off-topic disruption after maintainers ask for focus.
- Pressuring others to bypass security, licensing, or safety review.

## Enforcement

Maintainers may remove comments, close issues, reject contributions, or restrict
participation when behavior violates this code of conduct. Enforcement should be
proportional, documented when practical, and focused on keeping the project safe
and productive.

## Reporting

Report conduct concerns to the project maintainers through the repository issue
or security contact configured on GitHub. If the concern involves a maintainer,
contact another maintainer directly when available.

Reports should include:

- what happened;
- where it happened;
- who was involved;
- any links, screenshots, or logs that help review the issue.

## Scope

This code applies to project spaces, including issues, pull requests,
discussions, documentation, and project-related communication.

## Attribution

This code is based on common open-source community guidelines and adapted for
the `vCUDA-kernel` project.
